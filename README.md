# DSA Pattern Blueprints

https://nish616.github.io/DSA-cheat-sheet/

A comprehensive interview field guide for **SDE 2 / SDE 3 coding rounds**, featuring twelve battle-tested algorithmic patterns that cover ~90% of common interview problems.

## 📚 What's Inside

This is a beautifully designed reference guide featuring:

### 12 Algorithmic Patterns
1. **Sliding Window** - Contiguous subarrays & substrings with fixed/variable windows
2. **Two Pointers** - Sorted arrays & pairs optimization
3. **Fast & Slow Pointers** - Linked lists, cycles & sequence detection
4. **Merge Intervals** - Overlapping ranges & scheduling
5. **Binary Search** - Sorted data & O(log n) optimization
6. **Monotonic Stack** - Next/previous greater elements & span problems
7. **Heap / Top-K** - Priority-based selection & stream processing
8. **BFS** - Shortest paths in unweighted graphs & level-order traversal
9. **DFS & Backtracking** - Generate all permutations, subsets & island problems
10. **Dynamic Programming** - Counting ways, optimization & state transitions
11. **Topological Sort** - Dependency ordering & course scheduling
12. **Union-Find** - Connectivity, grouping & merge operations

### For Each Pattern
- **🔑 Trigger Keywords** - Words in problem statements that signal which pattern to use
- **🎨 Visual Models** - SVG diagrams showing how the pattern works
- **💻 Reference Implementation** - Clean, commented code with time/space complexity
- **🎯 LeetCode Problems** - High-signal problems to practice and drill

## 🚀 Getting Started

Simply open `index.html` in your browser to view the interactive guide. The page features:

- **Decision Map** - Quick pattern matching based on problem keywords
- **Navigation sidebar** - Easy jumping between patterns
- **Responsive design** - Works on desktop and mobile devices
- **Dark theme** - Blueprint-inspired UI optimized for readability

## 🎯 How to Use

1. **Read a problem statement** in a coding interview
2. **Identify trigger keywords** and consult the decision map
3. **Navigate to the relevant pattern** blueprint
4. **Review the visual model** to understand the approach
5. **Reference the implementation** for coding syntax
6. **Practice with LeetCode problems** to solidify understanding
